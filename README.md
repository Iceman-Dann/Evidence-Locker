## The Solution

EvidenceLocker is a **litigation-preparation workflow**, not a chatbot. It moves a user from a raw complaint to a formal case packet through four structured stages:

```mermaid

    A(["рҹ“қ Raw complaint\nplain English"]) --> B["вҡ–пёҸ Stage 1\nLegal framing\n5 parallel Gemini calls"]
    B --> C["рҹ”Қ Stage 2\nEvidence analysis\nAI checklist + strategy"]
    C --> D["рҹ“Һ Stage 3\nExhibit Intelligence\nGemini Vision per file"]
    D --> E(["рҹ“Ұ Stage 4\nFormatted packet\n10-page PDF export"])

    style A fill:#C41E1E,color:#fff,stroke:none
    style E fill:#2D7A3A,color:#fff,stroke:none
```

Most tools stop at Stage 2. EvidenceLocker builds Stages 3 and 4 вҖ” the ones that make a case **credible** to an opposing party.

```mermaid
sequenceDiagram

    User->>Intake: Describe dispute В· category В· jurisdiction
    Intake->>LS: Save normalized case payload

    Loading->>LS: Cache full analysis
    Loading->>Results: Redirect when ready

    User->>Results: Upload screenshots В· photos В· PDFs
    Results->>Results: SHA-256 fingerprint via Web Crypto API
    Results->>IDB: Store raw blobs in EvidenceLockerDB

    end

    Results->>Gemini: Synthesis вҖ” exhibit index + claim matrix + gaps + addendum

    Results->>LS: Cache packet in manifest

    User->>PDF: Download
    PDF-->>User: Cover + 5 analysis pages + 4 exhibit pages = 10 pages
```
